Merge conflicts

Two people edited the same lines. Git needs you to decide the final version.

Objective

Resolve a merge conflict and finish the merge safely.

What a conflict means

Git could not automatically combine changes because the same part of a file changed in different commits.

VS Code (recommended)

  1. Pull or merge and wait for VS Code to show conflicts in Source Control.
  2. Open the conflicted file → VS Code shows the merge editor options.
  3. Choose the correct result (keep yours, keep theirs, or combine).
  4. Save the file.
  5. Stage the resolved file(s) and commit the merge.

Command Line (fallback)

  1. See which files are conflicted:
git status
  1. Edit the file and remove the conflict markers:
<<<<<<< HEAD
your version
=======
their version
>>>>>>> branch-name
  1. Stage and commit:
git add path/to/conflicted-file
git commit

GitHub

  • In a Pull Request, GitHub may allow “Resolve conflicts” in the browser for simple cases.
  • For anything non-trivial, resolve locally in VS Code and push the result.

Check you succeeded

  • git status no longer lists unmerged paths.
  • Your code runs/tests still pass after the merge.

Common issues

  • Accidentally left conflict markers in file → search for <<<<<<< and remove them.
  • Merged but program breaks → re-open the conflict file and re-check logic/indentation.
