Daily workflow: edit → stage → commit → push

The routine you’ll repeat in every project.

Objective

Save your work as small commits and back it up to GitHub.

Command Line

  1. See what changed:
git status
  1. Review changes (optional but recommended):
git diff
  1. Stage the files you want in the next commit:
git add path/to/file.py
# or stage everything changed:
git add .
  1. Commit with a clear message:
git commit -m "Implement login validation"
  1. Push to GitHub:
git push

VS Code

  1. Open Source Control.
  2. Review file diffs by clicking changed files.
  3. Stage files using the + icons.
  4. Write a commit message → Commit.
  5. Sync/Push (depending on your VS Code settings).

GitHub

  • Open your repo page → you should see your newest commit at the top.
  • Use “History” to confirm your commits are arriving.

Check you succeeded

  • git status says “working tree clean”.
  • GitHub shows the new commit and updated files.

Common mistakes

Committing too much at once

Prefer small commits: one change, one message.

Forgetting to push

A commit on your laptop is not a backup until it’s on GitHub.
